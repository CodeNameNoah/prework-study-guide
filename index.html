Prework Study Guide

HTML

  • Each HTML element performs a unique role, however, two elements are critical to understanding how the structure works
  • The head element contains information or metadata which assists in allowing the browser to render the page correctly. Elements included within the head are not visible when a user looks at a webpage
  • The body element represents the visible content shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The border property assigns a border with a with of a specified px of your choosing.
  • The box-shadow CSS property adds shadow effects around an element's frame. Multiple effects can be separated by commas.
  • A period that precedes any "card" becomes designated as a class selector. Any HTML element assigned to the class card will gain respective property assignments from the CSS.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git status: show modified files in working directory, staged for your next commit
  • git add [file]: add a file as it looks now to your next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in working directory
  • git branch: list your branches. a * will appear next to the currently active branch
  • git branch [branch-name]: create a new branch at the current commit
  • git merge [branch]: merge the specified branch's history into the current one
  • git init: initialize an existing directory as a Git repository
  • git clone [url]: retrieve an entire repository from a hosted location via URL
